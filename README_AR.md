# دليل تشغيل مشروع EasyParking (للمطورين القادمين من PHP)

أهلاً بك! بما أنك معتاد على HTML/CSS/PHP، فإن هذه التقنيات الجديدة قد تبدو غريبة قليلاً، لكنها في الواقع تقوم بنفس العمل بطريقة مختلفة وأكثر حداثة. إليك الشرح والمقارنة:

## 1. مقارنة سريعة (المفاهيم)

| المفهوم | في PHP (القديم) | في Next.js (الجديد) |
| :--- | :--- | :--- |
| **الصفحات** | ملفات مثل `index.php`, `about.php` | ملفات داخل مجلدات: `app/page.tsx`, `app/about/page.tsx` |
| **HTML** | تكتبه مباشرة داخل ملف `.php` | نكتبه داخل دوال (Components) باستخدام صيغة تشبه HTML تسمى **JSX** |
| **قواعد البيانات** | `mysqli_query` أو `PDO` | نستخدم مكتبة **Supabase Client** (تشبه التعامل مع API) |
| **تحديث الصفحة** | يجب عمل Reload للصفحة لترى البيانات الجديدة | الصفحة **تتحدث تلقائياً** (Real-time) دون إعادة تحميل |

## 2. هيكلة المشروع (أين ملفاتي؟)

- **`app/`**: هذا هو المكان الأساسي. كل مجلد بداخله يعتبر "رابط" في الموقع.
    - `app/login/page.tsx` = صفحة تسجيل الدخول `yoursite.com/login`.
    - `app/dashboard/page.tsx` = صفحة لوحة التحكم.
- **`components/`**: قطع الغيار. بدل أن تكرر كود الـ Sidebar في كل صفحة (כما كنت تفعل في `include('header.php')`)، نكتبه مرة واحدة هنا ونستدعيه.
- **`public/`**: للصور والملفات الثابتة.

## 3. كيف تشغل المشروع (خطوة بخطوة)

### الخطوة الأولى: إعداد قاعدة البيانات (Supabase)
1. اذهب لموقع [Supabase](https://supabase.com) وأنشئ مشروعاً جديداً.
2. اذهب إلى **SQL Editor** في القائمة الجانبية.
3. انسخ محتوى الملف `schema.sql` الموجود في مشروعك وضعه هناك، ثم اضغط **Run**. هذا سينشئ الجداول.
4. اذهب إلى **Project Settings** -> **API**.
5. ستحتاج إلى مفتاحين: `Project URL` و `anon` (public) key.

### الخطوة الثانية: ربط المشروع
1. في جهازك، افتح الملف المسمى `.env.local`.
2. الصق الروابط التي أخذتها من Supabase:
   ```env
   NEXT_PUBLIC_SUPABASE_URL=رابط-المشروع-هنا
   NEXT_PUBLIC_SUPABASE_ANON_KEY=المفتاح-هنا
   ```

### الخطوة الثالثة: التشغيل
افتح "الطرفية" (Terminal) واكتب الأمر التالي:

```bash
npm run dev
```

الآن افتح المتصفح على الرابط: `Create Next App` (عادة يكون `http://localhost:3000`).

---

## 4. نصيحة "معلم لطالب"
في البداية، ركز فقط على مجلد `app`. حاول تغيير النصوص في `page.tsx` وسترى التغيير فوراً في المتصفح. لا تقلق بشأن التعقيدات الأخرى الآن. Next.js يعطيك "إطار عمل" جاهز بحيث لا تضطر لبناء كل شيء من الصفر كما في PHP الخام.
